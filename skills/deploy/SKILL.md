# Deployment Workflow

當我明確要求你「幫我部署」或表示要發佈專案時，請遵守以下流程：

1. **部署指令與監聽機制**：
   在協助我完成 Git Push（觸發 CI/CD）之後，請務必提供監聽指令，讓我能夠即時查看 GitHub Actions 的部署狀態。就像在技術筆記網站中一樣，請提供：
   ```bash
   npx ghaction-lis
   ```
   *(註：若專案有特殊的連鎖部署流程，可依據該專案特性加上 `--chain` 等參數)*

2. **安全防呆機制**：
   依循全域終端機安全規範，請將上述指令放在 Markdown 區塊中，**交由我手動複製執行**。除非我明確指示「你直接幫我跑」，否則絕對不可以主動執行部署或監聽指令。
