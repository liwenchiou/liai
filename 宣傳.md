最近在用 AI 輔助開發時，遇到了一些痛點，所以自己動手整理了一套工作流，想說來跟社群交流分享一下～

不知道大家最近在用 AI 寫 Code 時，會不會也有這些困擾：
1. **換模型就失憶**：遇到難解的 Bug 想切換到另一個模型（例如從 Claude 切換到 Gemini 或 OpenAI），結果剛講好的架構規範又要重教一次。
2. **AI 有時很愛「加戲」**：明明一個 HTML5 原生標籤就能搞定的功能，它非要幫你裝一堆套件、自己造輪子寫一大串。
3. **每次開新專案都要重新教育**：總是要重複提醒它三層架構、設計風格，不然專案一大架構就很容易跑掉。
4. **資安隱患**：網路上隨便抄的 `.cursorrules` 可能藏有惡意腳本，或是不小心讓 AI 把機密金鑰推上遠端。

為了解決這些問題，我把常用的機制整理成了一個可以「跨專案共用」且「跨模型持久記憶」的配置檔，叫做 **🧠 LIAI (全域 AI 協作大腦)**。
只要在任何新專案透過軟連結 (Symlink) 掛載它，AI 就會自動載入這些我整理好的預設習慣。

✨ **最大賣點與核心功能：**
- 💾 **切換模型保持記憶**：因為所有的大腦規範都實體化存在 `.agents` 目錄中，不管你今天切換成哪一家的 AI 模型，只要它讀取到這個目錄，就會瞬間變成同一個最懂你的資深工程師，**徹底解決換模型就失憶的痛點！**
- 🥸 **PM 視角 (`pm-skills`)**：在動手寫 Code 前，先引導釐清需求，產出 PRD 與 User Story。
- 👴 **極簡資深工程師 (`lazy-senior`)**：落實 YAGNI 哲學，防範 AI 過度設計。能用 1 行解決的絕不寫 10 行；Code Review 時也能幫忙挑出冗餘程式碼。
- 🛡️ **Wiz 企業級資安 (`backend-api`)**：參考了 Wiz 官方釋出的 OWASP Top 10 防禦基準，加上了白名單驗證、防 SQL 注入、自動過濾報錯等後端防禦性設定。
- 🎨 **Vercel 美學 (`frontend-design`)**：預設套用現代極簡的黑白 UI 風格，減少公版設計的廉價感。

🚨 **安全防護機制**
因為資安真的很重要，這套工作流裡也整合了 NVIDIA 開源的 `SkillSpector`，用來在載入第三方規則前進行強制掃描。同時也加了 Git 提交前的資安快篩，只要偵測到 `.env` 或 Hardcode 金鑰想偷跑，就會中斷提示。

這目前是我自己日常開發的主力工作流，如果大家剛好也有類似的困擾，或許可以參考看看，或是 Fork 回去改造成最適合自己的版本～！👇
